# 품질 점수표

이 문서는 변경 사항이 머지 가능한 수준인지 판단하기 위한 기준이다.

## 머지 게이트

머지 가능한 변경은 아래 조건을 모두 충족해야 한다.

- 선언한 범위 안에서 동작이 맞아야 한다
- 해당 변경에 맞는 최소 합리적 검증이 실행되어야 한다
- 동작이나 워크플로우가 바뀌었다면 문서도 함께 갱신되어야 한다
- 리스크와 생략한 검증은 PR에 명시되어야 한다

## 점수표

각 항목을 `pass`, `watch`, `fail` 중 하나로 평가한다.

| 항목 | 확인 내용 |
| --- | --- |
| 제품 정확성 | 기능이나 수정이 의도한 동작과 맞는가 |
| 코드 가독성 | 다른 사람이나 에이전트가 변경 경계를 빠르게 찾을 수 있는가 |
| 검증 상태 | 필요한 명령과 브라우저 확인이 실행되었는가 |
| 문서 정합성 | 필요한 문서와 실행 계획이 갱신되었는가 |
| 운영 안전성 | CI, 배포, 콘텐츠 작성 흐름이 유지되는가 |

## 기본 기대치

- UI 변경: 브라우저 확인과 `pnpm verify`
- 콘텐츠 전용 변경: `pnpm docs:check`와 `pnpm build`
- 워크플로우 변경: `pnpm verify`와 워크플로우 리뷰

## 실패 처리 원칙

- `fail`: 머지하지 않는다
- `watch`: PR에 명시적 메모와 후속 담당자가 있을 때만 머지한다
- `pass`: 추가 조치가 필요 없다
