# 관측 가능성 가이드

## 목표

너무 이른 시점에 무거운 인프라를 붙이지 않으면서도, 사람과 에이전트가 시스템 상태를 쉽게 확인할 수 있게 유지한다.

## 현재 신호

- `pnpm verify`의 로컬 명령 출력
- Next.js 빌드 출력
- CI와 배포용 GitHub Actions 로그
- UI 변경 시 브라우저 기반 시각 점검

## 최소 관측 규칙

- 모든 실패는 재현 가능한 명령, 라우트, 또는 워크플로우 단계로 연결되어야 한다
- PR에는 무엇을 확인했고 무엇을 생략했는지 적어야 한다
- 실행 계획에는 실패 증상과 선택한 수정 방향이 남아 있어야 한다

## 향후 추가 후보

- `docs/generated/`에 빌드 요약 리포트 생성
- 콘텐츠 검증 리포트 추가
- 주요 라우트의 스크린샷 기준선 추가
- 간단한 업타임 또는 배포 상태 배지 추가
