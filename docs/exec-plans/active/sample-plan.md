에이전트가 구현, 검증, 문서화, 배포를 더 적은 모호성으로 수행할 수 있도록 KeepLog의 가볍지만 지속 가능한 하네스 엔지니어링 기준선을 만든다.

## 범위

- 루트 수준 탐색 문서 추가

- 문서 품질 검사 추가
- PR 템플릿과 GitHub 워크플로우 추가
- Next.js export 동작을 GitHub Pages 배포와 맞추기

## 비목표

- 완전한 end-to-end 테스트 스위트 구축
- CMS 연동
- 대규모 UI 리디자인

## 현재 이해

- 앱 코드는 `blog/`에 있다
- Markdown 게시물은 `post/`에 있다
- 배포 대상은 GitHub Pages다
- 기존 Codex 가이드는 이미 `blog/.codex/` 아래에 있다

## 리스크

- GitHub Pages base path를 잘못 다루면 정적 자산 경로가 깨질 수 있다
- 계획과 함께 문서를 갱신하지 않으면 드리프트가 생길 수 있다
- 앞으로 서버 전용 기능이 추가되면 정적 export 전제가 깨질 수 있다
