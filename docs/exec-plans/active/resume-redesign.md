## 목표

첨부된 레퍼런스 이미지를 기준으로 `blog/src/app/resume/page.tsx`가 렌더링하는 소개 페이지를 전면 재구성한다. 기존 분절형 이력서 레이아웃을 프로필 헤더, 핵심 강점, 커리어 서사, 프로젝트, 기술 기반, 경험 연대표가 연결된 문서형 포트폴리오로 바꾼다.

## 범위

- `blog/src/features/resume/`의 데이터 구조 재설계
- 소개 페이지 섹션 구조와 스타일 재구성
- 레퍼런스 이미지 기준의 주요 텍스트 반영
- 반응형 레이아웃 및 브라우저 확인

## 비목표

- 새 백엔드 연동
- 블로그 또는 홈 화면 리디자인
- 자동 스크린샷 테스트 도입

## 현재 이해

- 현재 `resume` 페이지는 `Experience / Projects / Education / Certification / Awards / Activity` 순의 섹션형 타임라인이다
- 레퍼런스는 상단 프로필 요약, 강점 태그, 상세 커리어 기술, 프로젝트, 기술 기반, 경험 타임라인까지 포함하는 더 촘촘한 정보 구조를 사용한다
- 구현은 `blog/src/features/resume/` 안에 국한되어 있어 기능 경계를 유지한 채 재구성할 수 있다

## 리스크

- 텍스트량이 많아 모바일에서 밀도가 과해질 수 있다
- 레퍼런스 이미지의 정보를 잘못 축약하면 의도와 다른 이력서가 될 수 있다
- 기존 섹션 컴포넌트를 무리하게 재사용하면 정보 구조가 오히려 어색해질 수 있다
