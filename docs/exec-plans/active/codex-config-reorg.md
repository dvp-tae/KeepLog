## 목표

`blog/.codex`에 섞여 있는 Codex 설정, 저장소 전반 지침, 앱 전용 지침, 참조 문서를 현재 KeepLog 저장소 위계에 맞게 재배치한다.

## 범위

- 기존 `.codex` 파일의 역할을 분류한다
- Codex 런타임 설정 진입점을 저장소 루트 `.codex/`로 옮긴다
- 비실행성 참조 문서를 `docs/` 체계로 이동한다
- 변경된 위치에 맞춰 경로 참조와 상위 문서를 갱신한다

## 비목표

- 앱 기능 동작 변경
- 디자인 시스템 자체 재작성
- 새로운 Codex 스킬이나 에이전트 추가

## 현재 이해

- 기존 Codex 관련 파일은 모두 `blog/.codex/` 아래에 모여 있었다
- 그중 `config.toml`, `agents/`, `skills/`는 저장소 루트에서 찾는 편이 자연스럽다
- `하네스엔지니어링.md`는 실행 설정이라기보다 참고용 생성 프롬프트에 가깝다
- 일부 스킬/에이전트 문서는 `blog/codex/...`처럼 잘못된 경로를 가리키고 있었다

## 리스크

- 경로 이동 후 문서 참조가 일부 누락될 수 있다
- Codex가 특정 상대 경로를 기대한다면 스킬 문구를 함께 갱신해야 한다
- 상위 저장소 문서가 이전 위치를 계속 안내하면 이후 작업자가 혼란을 겪을 수 있다
