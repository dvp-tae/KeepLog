# 실행 계획 운영 가이드

실행 계획은 단순하지 않은 작업을 제어하기 위한 기본 문서다.

## 계획이 필요한 경우

다음에 해당하면 계획을 만들거나 갱신한다.

- 여러 파일이나 여러 서브시스템에 걸친 작업
- UI 구조나 정보 구조를 바꾸는 작업
- 빌드, CI, 배포, 작성 워크플로우를 수정하는 작업
- 원인이 즉시 보이지 않는 버그를 수정하는 작업
- 새 의존성이나 장기 규칙을 도입하는 작업

## 디렉터리 규칙

- `active/`: 진행 중, 보류 중, 또는 다음 구현 사이클에 바로 이어질 작업
- `completed/`: 완료되었고 최종 검증 상태와 결정 요약이 남아 있는 작업

## 계획 템플릿

각 계획에는 다음 항목이 포함되어야 한다.

1. 헤더: 제목, 상태, 담당자, 마지막 갱신일
2. 목표
3. 범위
4. 비목표
5. 현재 이해
6. 리스크
7. 단계
8. 검증
9. 결정 로그
10. 후속 작업

## 생명주기

1. `active/`에 계획을 만든다
2. 구현 방향이 크게 바뀌기 전에 문서를 갱신한다
3. 무엇을 검증했고 무엇을 생략했는지 남긴다
4. 변경이 머지되거나 의도적으로 중단되면 안정적인 요약과 함께 `completed/`로 옮긴다
