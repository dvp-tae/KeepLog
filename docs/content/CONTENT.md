# 콘텐츠 가이드

## 범위

콘텐츠는 `post/`의 Markdown으로 저장되고 `blog/` 앱에서 렌더링된다.

## 규칙

- 파일명은 slug를 결정하므로 안정적으로 유지한다
- 프론트매터는 `title`, `description`, `date`, `tags`, `draft` 형식을 일관되게 사용한다
- 설명 누락이나 잘못된 날짜 형식은 무시하지 말고 품질 문제로 수정한다
- 콘텐츠 리팩터링 중 draft가 의도치 않게 공개되지 않도록 한다

## 검증

- 콘텐츠 수정 후에는 빌드를 수행한다
- 포맷이나 Markdown 처리 방식이 바뀌면 실제 렌더링 결과도 확인한다
