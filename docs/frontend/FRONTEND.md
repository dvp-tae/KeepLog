# 프론트엔드 가이드

## 범위

이 문서는 `blog/`의 Next.js 앱에 적용되는 기본 프론트엔드 규칙을 정의한다.

## 구조 원칙

- 기능 코드는 `blog/src/features/` 아래에 둔다
- 공용 프리미티브는 `blog/src/components/`에 둔다
- 공용 스타일과 토큰은 `blog/src/styles/`에 둔다

## 규칙

- 새로운 범용 추상화를 만들기 전에 기존 feature 내부 수정으로 해결 가능한지 먼저 본다
- 임의의 스타일 상수보다 Emotion과 기존 테마 토큰 사용을 우선한다
- 시각 효과보다 콘텐츠 가독성을 먼저 지킨다
- 서버 컴포넌트와 클라이언트 컴포넌트 경계를 의도적으로 유지한다
- UI에 영향이 있는 작업은 데스크톱과 모바일 레이아웃을 브라우저에서 확인한다

## 변경 체크리스트

- 기능 경계가 여전히 명확한가
- 스타일이 해당 기능 또는 프리미티브 근처에 위치하는가
- 타이포그래피와 여백이 제품 톤과 맞는가
- 불필요한 클라이언트 상태를 늘리지 않았는가
