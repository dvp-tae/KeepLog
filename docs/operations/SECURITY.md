# 보안 가이드

## 범위

KeepLog는 현재 정적 콘텐츠 중심 웹 앱이지만 기본 보안 규칙은 동일하게 적용된다.

## 규칙

- 시크릿과 개인 토큰은 절대 커밋하지 않는다
- 배포에 필요한 환경별 값은 소스 파일이 아니라 GitHub 설정에 둔다
- 렌더링 규칙 검토 전까지 Markdown은 신뢰되지 않은 입력으로 취급한다
- 새 의존성은 추가 전에 검토한다. 특히 콘텐츠 처리와 빌드 시점 패키지를 주의한다
- 명확한 제품 요구가 없으면 외부 스크립트 태그나 서드파티 임베드를 추가하지 않는다

## 리뷰 체크리스트

- 이 변경이 새로운 입력 표면을 만들었는가
- Markdown 또는 HTML 렌더링 방식을 바꾸는가
- 의존성, 시크릿, 외부 서비스를 추가하는가
- 발행 또는 배포 권한 범위에 영향을 주는가
